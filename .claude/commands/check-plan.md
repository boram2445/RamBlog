이 대화에서 방금 완료한 작업을 `docs/roadmap1/week*.md` 체크리스트에 반영한다.
사용자가 추가 힌트를 전달했으면 `$ARGUMENTS` 에 있으니 매칭 시 우선 참고한다.

## 1단계 — 직전 작업 식별

대화 컨텍스트에서 아래 정보를 추출한다.

- 이번 대화에서 생성/수정한 파일 경로
- 추가한 컴포넌트·훅·서비스·API 함수명
- 기타 사용자가 "완료"·"끝남"·"됐다"고 언급한 작업

위 목록이 비어 있으면 "이번 대화에서 식별된 완료 작업이 없습니다." 를 출력하고 종료한다.

## 2단계 — 주차 파일 매칭

`docs/roadmap1/week*.md` 를 glob 으로 모두 찾아 Read 한다 (RamBlog 는 `week1.md`~ 로 계속 확장됨).

각 파일에서 **미체크 항목**만 후보로 추린다.

- Day 상세표(`| # | 할 일 | ✓ |`)에서 ✓ 열이 `[ ]` 인 행

1단계에서 추출한 파일 경로·컴포넌트명·함수명을 키워드로 삼아 후보 행의 "할 일" 서술과 매칭한다.
확신이 낮은 항목은 제외한다 — **잘못 체크하는 것보다 빠뜨리는 게 낫다.**

## 3단계 — Preview 출력 (Edit 호출 금지)

매칭 결과를 아래 표 형태로 출력한다.

| 파일 | Day/# | 할 일 (앞 20자) | 변경 후 | 근거 |
|------|-------|----------------|---------|------|

종속 갱신도 같은 표에 포함한다.

- 어떤 Day 상세표의 모든 `[ ]` 가 `[x]` 로 채워지면 → 상단 요약 표(`| D | 작업 |`)의 그 Day 행 **D 셀 앞에 `✅` 접두**를 추가 등재. (요약 표엔 상태 열이 없으므로 완료 표기는 ✅ 접두가 유일한 수단)

매칭이 0건이면 "매칭된 항목이 없습니다. `$ARGUMENTS` 에 작업 키워드를 직접 적어 다시 시도해 보세요." 를 출력하고 종료한다.

## 4단계 — 승인 대기

```
위 변경을 적용할까요? (y / 제외할 항목 지시)
```

한 줄만 출력하고 대기한다. 사용자가 일부 제외를 요청하면 표를 수정해 재출력하고 다시 대기한다.

## 5단계 — 적용

승인 후 각 매칭 항목에 대해 Edit 으로 치환한다.

치환 규칙:
- Day 상세표 ✓ 셀: `| [ ] |` → `| [x] |`
- 요약 표 Day 행: `| 41 |` → `| ✅ 41 |` (D 값 앞에 `✅ ` 접두. 서브데이 `41b` 도 동일)

모든 Edit 완료 후 아래 형식으로만 출력한다.

```
체크 완료: N건
- docs/roadmap1/weekX.md
```

## 가드레일

- `docs/roadmap1/week*.md` 외의 파일은 절대 수정하지 않는다.
- git 커밋, 패키지 설치, 다른 슬래시 커맨드 호출 금지.
- 매칭 확신도가 낮으면 preview 단계에서 "후보 부족 — 수동 지정 필요" 라고 표시하고 체크하지 않는다.
